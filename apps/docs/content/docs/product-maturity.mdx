title: 产品成熟度路线图
description: WeMail 从可用到可信赖的 8 个成熟产品方向、验收标准和后台总览说明
---

WeMail 的成熟化目标分成 8 个方向。系统设置页的「成熟度总览」会把这些方向聚合成管理员可见的状态面板，用来追踪当前能力、证据和下一步动作。

## 8 个方向

| 方向 | 成熟后的表现 |
| --- | --- |
| 可观测性和运维后台 | 管理员能看到系统诊断、最近错误、投递失败、关键链路健康和告警入口。 |
| 用户安全与风控 | 已支持 API Key scope、设备会话列表、会话踢出、登录历史、审计概览、邀请码有效期、批量生成、目标角色和限流策略可视化。 |
| 邮件产品体验 | 已支持发件人、主题、日期、附件、提取类型筛选，邮件详情包含原文、附件元信息、链接风险提示、保留倒计时和批量删除/导出。 |
| 通知与集成体系 | 已支持按目标、事件、邮箱、关键词和静默时间配置通知规则，Webhook 可签名、测试、查看投递日志、失败重试/重放，并预留 Slack、Discord、飞书、企业微信目标。 |
| 发信能力完善 | 发信身份、额度、失败、重试、退信、模板和 SPF/DKIM/DMARC 检查都可解释。 |
| 商业化和权限模型 | 支持套餐、团队、角色、共享邮箱、组织用量和组织审计。 |
| 文档和自助排障 | 独立域名、Telegram、Email Routing、Resend、OAuth 和常见错误都有场景化文档。 |
| 数据可靠性 | D1 migration、备份恢复、R2 清理、幂等处理、重复通知防护和定时任务记录可验证。 |

## 管理员总览

进入「系统设置」后，管理员会看到：

- 「系统诊断」：检查部署配置、Cookie、CORS、邮箱域名、Resend、Telegram 和 OAuth。
- 「运维中心」：聚合最近失败、Webhook/Telegram 投递异常、发信失败、过期待清理邮件和 D1/R2 绑定信号，带有可跳转的处理入口。
- 「成熟度总览」：展示 8 个方向的状态、完成方向数和每个方向的当前进度。
- 「个人设置 / 安全与会话」：用户可以查看当前账号的活跃会话、最近活跃时间、IP、到期时间，并撤销非当前设备。
- 「用户设置 / 治理概览」：管理员可以查看登录历史、失败原因、近期审计事件、限流策略和邀请码使用统计。
- 「用户设置 / 邀请与入场」：管理员创建邀请码时可以选择批量数量、目标角色和有效期，过期邀请码会被注册和 OAuth 新用户入场流程拒绝。
- 「邮件列表 / 高级筛选」：用户可以按发件人、主题、接收日期、附件状态和识别类型缩小消息列表，并对当前页消息执行批量删除或 JSON 导出。
- 「邮件详情」：详情页显示保留时间、原文、附件预览元信息和提取链接的 HTTPS 风险提示。
- 「Webhook / 通知规则」：用户可以创建目标为 Webhook、Telegram 或后续集成的通知规则，通过事件、邮箱、关键词和静默时间控制投递。
- 「发件箱 / 发信成熟度」：用户可以看到今日额度、失败重试策略、Return-Path 说明、发信身份、SPF/DKIM/DMARC checklist，并用内置模板快速新建发送。
- 「用户设置 / 套餐、团队与配额」：管理员可以查看免费/高级/团队套餐层级、默认组织空间、成员角色、共享邮箱、组织级用量和组织审计摘要。
- 「系统设置 / 可靠性后台」：管理员可以查看 D1/R2 绑定、migration 清单、清理任务运行记录、幂等入库策略和备份/恢复命令。

状态含义：

| 状态 | 含义 |
| --- | --- |
| 全部正常 | 当前方向已有可运营闭环，仍可继续优化。 |
| 有提醒 | 已有基础能力，但存在明显缺口或配置提醒。 |
| 需要处理 | 当前方向缺少关键能力，或配置会影响生产可用性。 |

## 验收方式

每个方向完成时至少要有：

1. 后端能力或数据证据。
2. 管理员或用户可见的产品入口。
3. 对应测试覆盖。
4. 文档站说明或排障清单。
5. 在「成熟度总览」中更新为可以解释的状态。

不要只看测试通过就认为成熟化完成。成熟产品的关键是出了问题能定位、能恢复、能解释。
